# Dynasty, Heirs & Intrigue Design

## Цель
Добавить долгосрочный династический контур: семья правителя, наследование власти, борьба наследников и связь с фракционной/внешней политикой.

## Основные сущности

### DynastyManager (singleton)
- Хранит `currentRuler` и `heirs`.
- Ведёт жизненный цикл: рождение, взросление, смерть, преемственность.
- Проверяет кризисы преемственности и запускает переход власти.
- Обновляет поддержку наследников на основе состояния фракций.

### Heir
- Базовые поля: имя, возраст, пол, портрет (пока опционально), статус жизни.
- Характеристики: харизма, жестокость, интеллект, военная подготовка, лояльность к правителю, зависть.
- Поддержка фракций: `Dictionary<FactionType, float> factionSupport`.
- Тайные черты: набор скрытых флагов (`SecretTrait`) для интриг.

## Геймплей

### Жизненный цикл
- Наследники стареют ежегодно.
- При достижении определённого возраста растёт риск смерти/болезней.
- Рождение новых наследников возможно через случайные события.

### Преемственность
- При смерти/уходе правителя `HandleRulerDeath()` выбирает нового правителя.
- Если есть явный сильнейший кандидат — власть переходит ему.
- Если поддержка размыта между несколькими кандидатами — кризис преемственности (штраф легитимности/лояльности).

### Интриги
- Низкая лояльность + высокая жестокость увеличивают риск заговора.
- Заговор инициирует выбор игрока: казнить / сослать / простить.

## Интеграции
- **FactionManager**: перерасчёт поддержки наследников и `GetPreferredHeir()`.
- **EventManager**: династические события (рождение, смерть, заговор).
- **GlobalMapManager**: династические браки с сверхдержавами.
- **SaveSystem**: полное сохранение данных династии.
- **UI**: окно династии, список наследников и их параметров.

## Баланс
- Династия даёт стабильность при грамотном управлении.
- Игнорирование наследников повышает риск дворцовых кризисов.
- Сильные наследники усиливают государство, но создают риск борьбы за власть.
