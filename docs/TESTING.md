# Система тестирования Caudillo Bay

Данный документ описывает структуру и правила написания автоматизированных тестов для проекта.

## Установка
Тестирование в Unity требует наличия следующих пакетов (уже добавлены в `manifest.json`):
- **Unity Test Framework**: Основной фреймворк для запуска тестов.
- **Code Coverage**: Инструмент для сбора метрик покрытия кода.

## Структура папок
- `Assets/Tests/EditMode/`: Юнит-тесты, не требующие запуска игровой сцены.
- `Assets/Tests/PlayMode/`: Интеграционные тесты, проверяющие работу компонентов в Unity Lifecycle.
- `Assets/Tests/TestHelpers/`: Вспомогательные классы и фабрики для создания тестовых данных.

## Запуск тестов в Редакторе
1. Откройте окно через **Tools -> Test & Coverage Runner**.
2. Используйте кнопки для запуска соответствующих типов тестов.
3. После запуска всех тестов с покрытием, нажмите **Open Coverage Report** для просмотра HTML-отчёта.

## Запуск через CLI
Для запуска тестов в пакетном режиме (используется в CI):
```bash
unity-editor -batchmode -runTests -testPlatform EditMode -testResults EditModeResults.xml -coverageOptions "generateHtmlReport;generateCoberturaReport"
```

## Метрики покрытия
Мы установили следующие минимальные пороги:
- **Общее покрытие (Line Coverage)**: >= 70%
- **Критические модули (Economy, Politics, Construction)**: >= 80%

Пайплайн в CI завершится ошибкой, если эти показатели не будут достигнуты.

## Добавление новых тестов
1. Создайте новый C# скрипт в соответствующей папке (`EditMode` или `PlayMode`).
2. Используйте атрибуты `[Test]` для обычных тестов и `[UnityTest]` для тестов, требующих корутины (PlayMode).
3. Используйте `TestHelper` для создания объектов, чтобы избежать зависимости от префабов в проекте.
